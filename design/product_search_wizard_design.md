# MarketGate 품목 검색 UI 재설계 문서

> **프로젝트:** MarketGate (화장품 수출 바이어 발굴 플랫폼)  
> **대상:** `ChatModePage.jsx` → 비채팅형 UI 대체  
> **목표:** 채팅 대신 단계별 위자드(Wizard) + 실시간 프리뷰 형태로 전환  
> **상태:** 설계 단계 (미반영)  
> **작성일:** 2026-05-03

---

## 1. 현재 문제 분석

### 1.1 ChatModePage 현재 구조
```
┌─────────────────────┬─────────────────────┐
│  🤖 채팅 히스토리    │  📊 바이어 리포트    │
│                     │                     │
│  💬 "K-뷰티 수출..."│  [리포트 상세]       │
│  🤖 "분석 중..."    │                     │
│                     │                     │
│  [퀵칩: K-뷰티]     │                     │
│  [건강식품] [패션]  │                     │
│                     │                     │
│  [입력창] [보내기]   │                     │
└─────────────────────┴─────────────────────┘
```

### 1.2 문제점
| 문제 | 영향 |
|------|------|
| 채팅 = 자유도 높음, 구조 없음 | 사용자가 "뭘 입력해야 하는지" 모름 |
| 입력 → 대기 → 결과 순차 | 진행 상태를 한눈에 못 봄 |
| 채팅 UI = 대화형 SaaS 느낌 | 수출/무역 전문 플랫폼의 신뢰도 하락 |
| 한 번에 하나만 볼 수 있음 | 이전 결과와 비교/탐색 어려움 |
| 모바일에서 채팅 + 리포트 동시 노출 불가 | 모바일 사용성 매우 낮음 |

---

## 2. 설계 방향: "스마트 필터 위자드"

**핵심 철학:** 채팅 대신 **단계별 조건 설정 → 실시간 미리보기 → 결과 탐색**의 구조화된 플로우

### 2.1 메타포 변경
| 현재 | 제안 |
|------|------|
| "비서에게 물어보는 느낌" | "전문 컨설턴트가 설문지를 받아보는 느낌" |
| 채팅 | 단계별 위자드 (Step Wizard) |
| 자유 입력 | 구조화된 선택 + 실시간 프리뷰 |
| 순차 대화 | 병렬 탐색 가능한 결과 대시보드 |

## 3. 화면 구조 (Wireframe)

### 3.1 전체 플로우
```
[Step 1: 품목 선택] ──→ [Step 2: 조건 설정] ──→ [Step 3: 결과 대시보드]
     │                       │                        │
     ▼                       ▼                        ▼
  카테고리 카드           국가/연도/수량           리스트 + 디테일 + 비교
  HS 코드 직접입력        실시간 필터 프리뷰        저장/보내기/인콰이어리
```

### 3.2 Step 1 — 품목 선택 (Product Selector)

```
┌─────────────────────────────────────────────┐
│  ← 뒤로가기              MarketGate         │
├─────────────────────────────────────────────┤
│                                             │
│  어떤 제품의 해외 바이어를 찾으시나요?         │
│                                             │
│  ┌────────┐ ┌────────┐ ┌────────┐          │
│  │ ✨     │ │ 💊     │ │ 👕     │          │
│  │ K-뷰티  │ │ 건강식품 │ │ K-패션 │          │
│  │ 330499 │ │ 210690 │ │ 611030 │          │
│  │ [선택됨]│ │ 준비중 │ │ 준비중 │          │
│  └────────┘ └────────┘ └────────┘          │
│                                             │
│  또는 직접 HS 코드 입력                      │
│  ┌─────────────────────────────┐           │
│  │  3 3 0 4 9 9               │ ← 6칸 분리 │
│  └─────────────────────────────┘           │
│           [다음 단계 →]                     │
│                                             │
└─────────────────────────────────────────────┘
```

**설계 상세:**
- **카테고리 카드:** 이미지 + 아이콘 + HS 코드 + 상태 배지
- **HS 코드 입력:** 6개 분리된 입력 칸 (각 칸 1자리 숫자) — 자동 포커스 이동
- **유효성 검사:** 실시간 HS 코드 유효성 체크 (숫자 6자리)
- **선택 시 피드백:** 카드 확대 + 테두리 강조 + 미니 설명 팝업

- **체크박스 필터:** 지역 필터 — 실시간 프리뷰 숫자 변경
- **슬라이더:** 무역액 필터 — 최소/최대 범위
- **실시간 프리뷰:** 조건 변경마다 API 없이 프론트엔드 추정치 표시 ("약 N개국, M명 바이어")
- **진행 단계 표시:** "2/3 단계" — 사용자에게 진행감 제공

## 4. 컴포넌트 상세 설계

### 4.1 `<ProductSelector />` (Step 1)
```typescript
interface ProductCategory {
  id: string;
  label: string;
  hsCode: string;
  icon: LucideIcon;
  imageUrl?: string;
  available: boolean;
  description: string;        // "스킨케어, 메이크업, 헤어케어 등"
  sampleProducts: string[];   // ["에센스", "크림", "선크림"]
}

interface Props {
  categories: ProductCategory[];
  onSelect: (category: ProductCategory) => void;
  onCustomHsCode: (hsCode: string) => void;
  selected?: ProductCategory;
}
```

**인터랙션:**
1. 카드 hover → 설명 툴팁
2. unavailable 카드 → grayscale + "준비중" 오버레이
3. HS 코드 6자리 입력 완료 → 자동으로 "다음" 버튼 활성화
4. 자주 쓰는 코드 → 로컬 스토리지 기억 (최근 3개)

### 4.2 `<ConditionBuilder />` (Step 2)
```typescript
interface SearchConditions {
  topN: 3 | 5 | 8 | 10;
  year: 2021 | 2022 | 2023;
  regions: Region[];
  minTradeValue: number;      // USD
  includeDiagnostics: boolean;
}

interface Props {
  product: ProductCategory;
  conditions: SearchConditions;
  onChange: (conditions: SearchConditions) => void;
  onBack: () => void;
  onSubmit: () => void;
  previewEstimate: {         // 프론트엔드 추정 (API 안 쏨)
    estimatedCountries: number;
    estimatedBuyers: number;
  };
}
```

**인터랙션:**
1. 조건 변경마다 `previewEstimate` 실시간 업데이트 (로컬 캐시 기반)
2. "분석 시작" 버튼 → 로딩 스피너 + "데이터 분석 중..."
3. 진행률 바: 0% → 품목 확인 → 조건 적용 → 국가 필터링 → 바이어 매칭 → 100%

## 6. 반응형 (Responsive) 설계

### 6.1 Desktop (>1024px)
- 3단계 모두 전체 화면
- Step 3: 좌측 필터 사이드바 + 우측 결과 그리드 + 하단 비교 바

### 6.2 Tablet (768–1024px)
- Step 3: 필터 → 상단 축소 바, 결과 2열 그리드
- 비교 → 드로어 형태 (하단 슬라이드 업)

### 6.3 Mobile (<768px)
- Step 1–2: 전체 화면, 세로 스크롤
- Step 3: 
  - 필터 → 상단 고정 토글 버튼 행
  - 결과 → 1열 카드
  - 상세 → 풀스크린 슬라이드
  - 비교 → 하단 시트 (Bottom Sheet)

## 10. A/B 테스트 아이디어

| 테스트 | 가설 | 지표 |
|--------|------|------|
| Step 1 카드 vs 드롭다운 | 이미지 카드가 직관적이라 전환율 높음 | Step 1 → 2 전환 |

| 비교 기능 유무 | 2개국 비교가 인콰이어리 전환 ↑ | 인콰이어리 생성률 |
