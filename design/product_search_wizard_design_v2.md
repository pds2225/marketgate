# MarketGate 프론트엔드 설계 문서 v2.0
# — 사업계획서 기반 AI 글로벌 수출 거래 플랫폼 UI/UX 설계

> **프로젝트:** MarketGate (AI 기반 글로벌 수출 거래 마켓플레이스)  
> **버전:** v2.0 — 사업계획서 반영  
> **목표:** 단순 정보 제공 → End-to-End 거래 플랫폼 전환  
> **수익 모델:** 크레딧 기반 (1💎 = ₩1,000) — 토스페이먼츠/PayPal 결제 + 파트너 연계 수수료  
> **상태:** 설계 단계 (미반영)  
> **작성일:** 2026-05-03

---

## 1. Executive Summary

### 1.1 사업계획서 핵심 반영

| 사업계획 요소 | 프론트엔드 반영 방식 |
|--------------|---------------------|
| **5단계 거래 플로우** | 메인 네비게이션 → 5개 스텝 위자드 |
| **과금 구조** | 1💎 = ₩1,000 — 단계별 크레딧 소진 (15/20/50/5/10) |
| **크레딧 투명화** | 각 단계별 크레딧 소모량 실시간 표시 + 원화 병기 |
| **TDD 3단계 검증** | 바이어 카드에 검증 뱃지 (활동성/신용/컨택) |
| **트리거 연계** | 계약 완료 → 관세/물류 파트너 자동 매칭 모달 |
| **파트너 Pool** | 관세사/물류사/무역에이전트 매칭 마켓플레이스 |
| **수익성 시뮬레이션** | Profit Sim 단계 — 단가/물류비/관세 자동 계산 |

### 1.2 키 메시지 변화

**기존:** "수출 가능성을 분석하고 맞는 바이어를 찾는 AI 무역 매칭 플랫폼"  
**→ 신규:** "AI가 거래까지 완료하는 글로벌 수출 거래 마켓플레이스"

## 2. 사용자 페르소나 & 여정 (JTBD)

### 2.1 Primary — "첫 수출 도전 중소기업 대표"
- **Job:** K-뷰티 제품으로 일본에 첫 수출하고 싶다
- **통증:** 어디에 팔아야 할지 모름, 바이어 진짜인지 못 믿음, 물류/관세 복잡
- **플랫폼 여정:** 무료 탐색 → 수익성 확인 → 검증된 바이어 매칭 → 컨택 → 계약 → 물류 연계

### 2.2 Secondary — "바이어 확장 중인 기존 수출기업"
- **Job:** 베트남/태국 등 신규 시장 진출
- **통증:** 기존 바이어 풀 고갈, 신규 바이어 검증 시간 소모
- **플랫폼 여정:** 빠른 시장 스캔 → 수익성 비교 → 다국가 동시 컨택 → 계약 체결

### 2.3 Tertiary — "무역 중개인/에이전트"
- **Job:** 고객사 대신 수출 대행, 수수료 받음
- **통증:** 바이어 발굴 = 수작업, 검증 = 경험에 의존
- **플랫폼 여정:** 다중 클라이언트 관리 → 자동화된 바이어 검증 → 대행 수수료 과금

- **성과 중심:** "바이어를 찾습니다"가 아닌 "계약까지 완료합니다"
